# Скрипты

Один общий скрипт на источник данных, а не по скрипту на клиента — разница между
клиентами только в логине/токене/номерах целей (в `.env` и `Клиенты/<клиент>/Цели/goals.md`),
структура запроса одна и та же.

Источник вдохновения/логики: `E:/PythonProjects/RedBird/google-cloud-jobs` (продовый
пайплайн в BigQuery для дашборда по всем клиентам). Здесь — облегчённая локальная версия
без BigQuery/Google Sheets, заточенная под этот вики-проект: снапшот кладётся прямо в
`Клиенты/<клиент>/Статистика/` и дальше анализируется агентом.

## Установка

```
pip install -r requirements.txt
```

## Настройка клиента

В `.env` (в корне вики, не коммитится) добавить блок с префиксом — коротким ключом
клиента без пробелов:

```
<KEY>_DIRECT_LOGIN=...
<KEY>_DIRECT_TOKEN=...
<KEY>_YM_COUNTER=...
<KEY>_YM_TOKEN=...
```

## Запуск

```
python direct_stats.py --client-key EGEMERLIN --client-folder "ЕГЭ Merlin" --days 30
python metrika_stats.py --client-key EGEMERLIN --client-folder "ЕГЭ Merlin" --days 30 --goals 123456,789012
```

Результат — TSV/CSV файл в `Клиенты/<клиент>/Статистика/` с диапазоном дат в имени.
Номера целей для `--goals` берутся из `Клиенты/<клиент>/Цели/goals.md`.
