# Как считается конверсия: Директ API vs Метрика API

## Проверенный факт

И Директ (`Conversions_<id>_AUTO` в отчётах Reports API), и Метрика считают цель
одинаково — **по визитам, а не по количеству срабатываний**:

- **Метрика**: "достижение цели" — момент выполнения условия, может случиться
  несколько раз за один визит. Но "целевой визит" — это визит, в котором цель
  достигнута один и/или более раз, то есть один визит даёт максимум одну
  единицу в счётчике целевых визитов, сколько бы раз условие ни сработало внутри
  визита (например, три отправки формы одним человеком за визит = один целевой
  визит, не три).
- **Директ**: `Conversions_<id>_AUTO` в документации определяется как
  "количество целевых визитов (конверсий)" — та же единица: визит, в котором
  цель достигнута хотя бы раз, а не сумма всех достижений.

Источники: [официальная документация Директ API — Reports/Report format](https://yandex.ru/dev/direct/doc/reports/report-format.html),
[документация Метрики по отслеживанию конверсий](https://yandex.ru/dev/metrika/doc/api2/practice/conversion.html).

## Практический вывод

Расхождение в цифрах между Директом и Метрикой по одной и той же цели за один
и тот же период **не объясняется** разницей "визит vs количество срабатываний"
— обе системы считают визиты одинаково. Если расхождение есть — причина в
чём-то другом: разница в атрибуции (окно 21 день, модель атрибуции), в разной
привязке визита к кампании/источнику ("Яндекс.Директ: не определено" и т.п.),
или в самих данных/выгрузке, а не в методологии подсчёта единицы конверсии.
