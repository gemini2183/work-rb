# Товарная галерея / упрощённый Мастер кампаний — недоступность в Campaigns API

## Проверенный факт

Мастер кампаний в интерфейсе Директа — не отдельный тип в API, а надстройка,
создающая на бэкенде обычные `TEXT_CAMPAIGN` или `UNIFIED_CAMPAIGN` (Единая
перфоманс-кампания). Для таких кампаний `Campaigns.get` работает нормально.

Но есть отдельный случай — **Товарная галерея / Товарный мастер** (упрощённый
интерфейс с автоподбором товаров/фида): конфигурация таких кампаний в Директе
**read-only и не отдаётся через `Campaigns.get` вообще**, ни при пустом
`SelectionCriteria`, ни с явным `Types` (проверяли `UNIFIED_CAMPAIGN` и
`SMART_CAMPAIGN`, оба — пустой результат), ни прямым запросом по конкретному
`Id`. Это не баг запроса и не проблема прав доступа токена — архитектурное
ограничение самого API для этого вида кампаний.

## Как распознать на практике

Признак: кампания видна и откручивается в `Reports API` (есть клики/расход/
конверсии по её `CampaignId`), но полностью отсутствует в ответе
`Campaigns.get` под тем же логином/токеном — включая явную попытку получить
её по конкретному `Id`. Если при этом у клиента ещё и не возвращаются все
кампании с ID новее определённого порога — это тоже может быть тем же
эффектом (несколько таких кампаний подряд, не одна отдельная аномалия).

## Что делать

- **Статистика** (расход, клики, конверсии) — брать из `Reports API` как
  обычно, там ограничений нет.
- **Настройки/стратегию/тип оплаты** — узнать можно только через веб-интерфейс
  Директа, программного доступа нет.
- Если по такой кампании расход выглядит аномально (например, много кликов
  при 0₽ в отдельные дни) — не спешить считать это багом данных: при
  стратегии "Оплата за конверсии" (`PAY_FOR_CONVERSION`) списание не
  привязано к кликам напрямую, расход по дням может быть неравномерным и
  подолгу нулевым. Это нормальное поведение стратегии, а не ошибка отчёта —
  но подтвердить тип стратегии можно только глазами в интерфейсе.
